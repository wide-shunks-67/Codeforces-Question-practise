<div class="header"><div class="title">C. Simons and Posting Blogs</div><div class="time-limit"><div class="property-title">time limit per test</div>2 seconds</div><div class="memory-limit"><div class="property-title">memory limit per test</div>256 megabytes</div><div class="input-file input-standard"><div class="property-title">input</div>standard input</div><div class="output-file output-standard"><div class="property-title">output</div>standard output</div></div><div><p>  </p><div class="epigraph"><div class="epigraph-text"><span class="tex-font-style-it">This life is like a TV show; we're swept along as the plotlines go.</span></div><div class="epigraph-source">— SHUN, <span class="tex-font-style-it"><a href="https://open.spotify.com/track/4lfS48CaivAm4QaimqbAPv">TRAP</a></span></div></div><p>There are $$$n$$$ blogs. The $$$i$$$-th blog mentioned $$$l_i$$$ users in order as an array $$$a_i=[a_{i,1},a_{i,2},\ldots,a_{i,l_i}]$$$.</p><p>You are going to post all $$$n$$$ blogs. Let us maintain a sequence $$$Q$$$ that describes the list of users you have recently mentioned. You need to perform the following operation <span class="tex-font-style-bf">exactly</span> $$$n$$$ times:</p><ul> <li> Choose an <span class="tex-font-style-bf">unposted</span> blog $$$i$$$ ($$$1\le i\le n$$$), then post it. This will cause the following operations for each $$$1\le j\le l_i$$$ in order: <ul> <li> If $$$a_{i,j}$$$ already exists in $$$Q$$$, then move $$$a_{i,j}$$$ to the beginning of $$$Q$$$. </li><li> Otherwise, insert $$$a_{i,j}$$$ at the beginning of $$$Q$$$. </li></ul> </li></ul><p>Find the lexicographically smallest$$$^{\text{∗}}$$$ $$$Q$$$ after all $$$n$$$ operations.</p><div class="statement-footnote"><p>$$$^{\text{∗}}$$$An array $$$x$$$ is lexicographically smaller than an array $$$y$$$ if and only if one of the following holds: </p><ul> <li> $$$x$$$ is a prefix of $$$y$$$, but $$$x \ne y$$$; or </li><li> in the first position where $$$x$$$ and $$$y$$$ differ, the array $$$x$$$ has a smaller element than the corresponding element in $$$y$$$. </li></ul> </div></div><div class="input-specification"><div class="section-title">Input</div><p>Each test contains multiple test cases. The first line contains the number of test cases $$$t$$$ ($$$1 \le t \le 1000$$$). The description of the test cases follows. </p><p>The first line contains a single integer $$$n$$$ ($$$1\le n\le 3000$$$) — the number of blogs.</p><p>Then $$$n$$$ lines follow, the $$$i$$$-th line starting with an integer $$$l_i$$$ ($$$1\le l_i\le 3000$$$), describing the number of users mentioned in the $$$i$$$-th blog, which is followed by $$$l_i$$$ integers $$$a_{i,1},a_{i,2},\ldots,a_{i,l_i}$$$ ($$$1\le a_{i,j}\le 10^6$$$) — the list of users mentioned in the $$$i$$$-th blog.</p><p>It is guaranteed that the sum of $$$n$$$ over all test cases does not exceed $$$3000$$$. </p><p>Denote $$$\sum\limits_{i=1}^n l_i$$$ as $$$L$$$. It is guaranteed that the sum of $$$L$$$ over all test cases does not exceed $$$3000$$$. </p></div><div class="output-specification"><div class="section-title">Output</div><p>Denote $$$m$$$ as the number of users mentioned in at least one blog. For each test case, output $$$m$$$ integers $$$Q_1,Q_2,\ldots,Q_m$$$ — the lexicographically smallest $$$Q$$$.</p></div><div class="sample-tests"><div class="section-title">Example</div><div
